🤖 Kaj je Claude Code?

Claude Code ni dopolnjevanje kode v urejevalniku — je agent, ki sam odpre terminal, prebere repozitorij in dokonča nalogo.

Claude Code je Anthropicovo agentno orodje za ukazno vrstico (CLI), ki teče neposredno v terminalu razvijalca. Namesto da bi predlagal le naslednjo vrstico kode, Claude Code samostojno brska po datotekah projekta, zaganja teste, popravlja napake, ustvarja gita commite in odpira pull requeste. Razvijalec mu poda nalogo v naravnem jeziku — "popravi bug v plačilnem modulu" ali "dodaj testno pokritost za auth flow" — agent pa sam poišče relevantne datoteke, naredi spremembe in preveri, ali rešitev deluje.

Ključna razlika v primerjavi s klasičnimi "copilot" orodji je avtonomija. Claude Code lahko verigo nalog izvede brez vmesnega potrjevanja vsake vrstice: prebere issue, naredi plan, implementira spremembo čez več datotek, požene test suite in poroča, kaj je naredil. Razvijalec ostane v vlogi nadzornika — pregleda diff, odobri ali popravi smer — ne pa tipkača vsake vrstice.

"Razlika med 'AI dopolnjevanjem kode' in 'agentnim kodiranjem' je razlika med predlogom naslednje besede in dejanskim opravljenim delom."

Za slovenske ekipe je to pomembno iz dveh razlogov:

  • Manjše ekipe lahko dosežejo več. Solo founder ali 2-3 člana lahko vzdržujejo obseg kode, za katerega bi prej potrebovali 5-6 razvijalcev.
  • Onboarding na velike, slabo dokumentirane repozitorije je hitrejši. Agent prebere celotno kodno bazo in razloži arhitekturo novemu sodelavcu ali strankinemu razvijalcu v minutah, ne dnevih.

🪽 Kaj je Hermes Agent?

Če je Claude Code zaprt, oprt na enega ponudnika in pripravljen "iz škatle", je Hermes Agent nasprotni pol: odprt, modularen in zgrajen za tiste, ki želijo nadzor nad vsako plastjo.

Hermes Agent izhaja iz družine odprtokodnih modelov Hermes (Nous Research), ki so prilagojeni (fine-tuned) za function calling, strukturiran izhod in agentne delovne tokove. Namesto enega zaprtega sistema gre za gradnik: razvijalec model namesti lokalno ali v lastnem oblaku, ga poveže s svojim orkestratorjem (npr. LangChain, AutoGen, lasten agentni loop) in zgradi agenta po meri — od code-review bota do avtonomnega DevOps agenta, ki spremlja produkcijo.

Za podjetja z občutljivimi podatki (zdravstvo, finance, javna uprava) je to ključna prednost: model lahko teče on-premise ali na SLAIF/EuroHPC infrastrukturi, brez da bi koda ali podatki kadarkoli zapustili Slovenijo. Cena je drugačna kompleksnost — Hermes ne pride z vgrajenim terminalskim vmesnikom kot Claude Code; razvijalec mora sam zgraditi agentni loop, orodja in varovalke (guardrails).

Pogosta zmota: "Odprtokodno pomeni brezplačno in enostavno." V praksi samostojno gostovanje (self-hosting) Hermesa zahteva GPU infrastrukturo, MLOps znanje in vzdrževanje — strošek se preseli iz naročnine v inženirski čas.

🛠️ Praktične uporabe za slovenske builderje

Pet konkretnih scenarijev, kako agentno kodiranje že danes prinaša denar slovenskim agencijam in solo developerjem.

#1Agencija dostavi 3× več projektov z enako ekipo
AgencijaB2B🏔️ Claude Code€€€€
Realen problem: Spletna agencija v Ljubljani s 4 razvijalci sprejme naročilo za WordPress-to-Next.js migracijo. Ocenjen čas: 6 tednov. Stranka plača po fiksni ceni — vsak dodaten teden pomeni izgubo marže.
💰 Koliko denarja rešiš: Če Claude Code skrajša implementacijski čas za 40 %, agencija isto ekipo uporabi za 1,5–2 dodatna projekta na četrtletje — brez novih zaposlitev. Pri povprečni vrednosti projekta 15.000 € je to 20.000–30.000 € dodatnega prihodka na ekipo letno.
Kako agent reši: Claude Code prebere staro WordPress kodno bazo, generira komponentno strukturo v Next.js, samodejno migrira vsebinske tipe in zažene regresijske teste. Razvijalec preveri diff in popravi robne primere namesto pisanja vsega od začetka.
🛠️ Tehnologije

Agent: Claude Code CLI povezan z repozitorijem prek git worktree za varno paralelno delo.
MCP strežniki: povezava na Figma (za design tokens) in Linear/Jira (za task tracking).
CI: GitHub Actions — agent požene teste pred vsakim commitom.

#2Solo founder vzdržuje produkt brez najema razvijalca
Micro-SaaSSolo🏔️ Claude Code€€€
Realen problem: Solo founder iz Maribora vodi micro-SaaS s 200 plačujočimi strankami. Vsak teden pride 5-10 bug reportov in feature requestov. Sam ne zmore vsega kodirati in obenem prodajati.
💰 Koliko denarja rešiš: Najem junior razvijalca stane 2.500–3.500 €/mesec bruto. Claude Code naročnina stane 20–200 €/mesec, odvisno od porabe — 10–15× ceneje za enak obseg vzdrževalnega dela.
Kako agent reši: Founder vsak bug report prilepi agentu kot nalogo. Agent poišče vzrok, napiše popravek, požene teste in pripravi pull request. Founder samo pregleda in potrdi merge — ostane mu čas za prodajo in podporo strankam.
#3Banka ali zavarovalnica zgradi interni code-review agent na Hermesu
EnterpriseRegulirano🏔️ Hermes / on-premise€€€€€
Realen problem: Finančna institucija ne sme pošiljati izvorne kode v zunanji API zaradi regulative (GDPR, interna politika o občutljivih podatkih). Hkrati želi izkoristiti agentno kodiranje za hitrejši code review.
💰 Koliko denarja rešiš: Code review v reguliranih panogah traja 2-4 ure na pull request zaradi varnostnih in skladnostnih pravil. Avtomatiziran prvi pregled skrajša to na 30 minut — pri 50 PR-jih na mesec je to 100+ ur prihranjenega seniorskega časa.
Kako agent reši: Hermes model, fine-tuniran na interna varnostna pravila podjetja, teče on-premise. Agent pregleda vsak pull request, preveri skladnost s coding standardi in OWASP pravili ter oceni tveganje, preden gre pred človeškega reviewerja.
🛠️ Tehnologije (Enterprise Stack)

Model: Hermes (fine-tune na Llama 3 ali Mistral osnovi) z function calling za klic internih API-jev.
Orkestracija: LangChain ali lasten agentni loop v Python.
Deploy: vLLM na SLAIF / EuroHPC ali lasten on-premise GPU cluster.
Integracija: Webhook iz GitLab/GitHub Enterprise ob vsakem pull requestu.

#4Freelancer avtomatizira dokumentacijo in onboarding novih strank
FreelanceB2B🏔️ Claude Code€€
Realen problem: Freelance razvijalec prevzame stranko z 3 leta staro, slabo dokumentirano kodno bazo. Razumevanje arhitekture pred prvim popravkom vzame 1-2 tedna neplačanega "raziskovanja".
💰 Koliko denarja rešiš: Skrajšanje onboardinga z 10 dni na 2 dni pomeni 8 dodatnih plačljivih dni na novo stranko — pri 400 €/dan je to 3.200 € prihranjenega/zasluženega časa na projekt.
Kako agent reši: Claude Code v nekaj minutah prebere celotno kodno bazo, generira arhitekturni povzetek, seznam tehničnega dolga in predlog prvih varnih popravkov. Freelancer ima dokument, ki bi ga sicer pisal teden dni.
#5Startup zgradi lastnega "support-to-code" agenta za hitro odzivnost
Vertical SaaSB2B🏔️ Claude Code + Hermes hibrid€€€€
Realen problem: Startup v rasti dobiva 20+ podpornih zahtevkov dnevno, od katerih je tretjina dejanski bug, ne vprašanje uporabnika. Podporna ekipa bug le posreduje razvijalcem, kar podvoji čas reševanja.
💰 Koliko denarja rešiš: Hitrejše reševanje bugov zniža churn. Pri SaaS s 50.000 €/mesec MRR že 1 odstotna točka manjšega churna pomeni 500+ €/mesec ohranjenega prihodka.
Kako agent reši: Lasten triage agent (na Hermesu, zaradi nadzora nad podatki podpornih ticketov) klasificira zahtevek kot bug/vprašanje/feature. Pravi bugi gredo avtomatsko v Claude Code, ki pripravi popravek in odpre pull request, še preden razvijalec sploh prebere ticket.

Splošni nasveti

  • Začni s Claude Code za interno produktivnost. Tudi regulirana podjetja lahko agenta uporabljajo na nekritičnem, javno dostopnem delu kode, dokler ne zgradijo on-premise alternative.
  • Vedno pusti človeka v zanki. Agent predlaga, človek potrjuje merge v produkcijo — to ni opcija, ampak varnostna zahteva.
  • Meri prihranjen čas, ne "kul faktor". Stranki ne prodajaš AI-ja, prodajaš hitrejšo dostavo in nižji strošek vzdrževanja.
  • Hibridni pristop pogosto zmaga. Claude Code za splošni razvoj, Hermes/odprt model za del, ki obravnava občutljive podatke.

🗺️ Roadmap: kako agente vpeljati v ekipo

Faza 1: Pilot (0–14 dni)

  • Izberi en nekritičen repozitorij. Internal tool, marketing stran ali manjši micro-servis — ne plačilni sistem.
  • Namesti Claude Code in zaženi 3-5 realnih nalog. Bug popravki, dodajanje testov, refaktoring.
  • Meri čas pred/po. To je tvoj dokaz vrednosti za vodstvo ali stranko.

Faza 2: Vpeljava v ekipo (14–60 dni)

  • Definiraj pravila igre. Kdaj agent lahko sam naredi commit, kdaj potrebuje review. Zapiši v CONTRIBUTING.md.
  • Poveži MCP strežnike. Linear/Jira, Figma, monitoring — agent naj ima enak kontekst kot razvijalec.
  • Usposobi ekipo za "agent management". Veščina ni več samo pisanje kode, ampak postavljanje natančnih nalog in pregledovanje diffov.

Faza 3: Skaliranje in regulirani primeri (60–180 dni)

  • Za občutljive podatke postavi Hermes/odprt model on-premise. Začni z eno nekritično agentno funkcijo (npr. code review).
  • Zgradi interno knjižnico promptov in agentnih nalog. To postane konkurenčna prednost agencije ali ekipe.
  • Cilj: 30-50 % hitrejša dostava na ponavljajočih se nalogah (migracije, bug fixi, testna pokritost), merjeno na realnih projektih.

🎯 Zaključek: prihodnost je agentna

Claude Code in Hermes Agent nista konkurenta v smislu "izberi enega" — predstavljata dva konca istega spektra: hitrost in kvaliteta iz škatle proti popolnemu nadzoru in prilagodljivosti. Slovenske ekipe, ki bodo zmagale v naslednjih dveh letih, ne bodo tiste, ki agentno kodiranje ignorirajo, ampak tiste, ki znajo izbrati pravo orodje za pravo nalogo — Claude Code za hitro dostavo, odprte modele tipa Hermes za primere, kjer podatki ne smejo zapustiti podjetja.

Za slovenski trg je to še posebej relevantno:

  • Primanjkljaj 13.000 ICT specialistov pomeni, da agentno kodiranje ni luksuz, ampak nujnost za ohranjanje konkurenčnosti.
  • SLAIF in EuroHPC infrastruktura omogočata, da tudi manjša slovenska podjetja gostijo odprte modele brez tujih oblačnih stroškov.
  • Regulirane panoge (banke, zdravstvo, javna uprava) dobijo pot do AI produktivnosti, ne da bi kršile zahteve o lokaciji podatkov.

Ne čakaj, da se odločiš med "popolnim" in "začetnim" pristopom. Namesti Claude Code danes na en nekritičen repozitorij, meri rezultate v dveh tednih in šele nato načrtuj, kje potrebuješ odprt, on-premise pristop.

Začni z enim agentom danes.

Najboljši čas za vpeljavo agentnega kodiranja je bil pred letom dni. Drugi najboljši čas je danes — odpri terminal in dodeli prvo nalogo.

Opomba: Claude Code je orodje podjetja Anthropic. Hermes je odprtokodna družina modelov, ki jo vzdržuje Nous Research. Imena in znamke so v tem zapisu uporabljena izključno v informativni in primerjalni vlogi.
